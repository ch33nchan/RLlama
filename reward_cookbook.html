RLlama Reward Shaping Cookbook (LLM RLF Focus)

This guide provides practical recipes for using the rllama.rewards framework to engineer sophisticated reward signals for Reinforcement Learning Fine-tuning (RLF) of Large Language Models (LLMs).

Core Concepts Recap

  • Reward Components (BaseReward): Define individual reward sources (preference scores, safety penalties, instruction adherence metrics, verbosity costs, etc.). Implement by inheriting rllama.rewards.base.BaseReward.
  • Reward Composer (RewardComposer): Aggregates raw values from components and combines them using dynamic weights. Handles optional normalization. Found in rllama.rewards.composition.
  • Reward Shaper (RewardShaper & RewardConfig): Manages how component weights change over training steps (curriculum learning, fading penalties). Configured via YAML/Python. Found in rllama.rewards.shaping.
  • Registry (rllama.rewards.registry): Maps string names to component classes for loading from YAML.
  • YAML Configuration: Declaratively define the entire reward strategy for reproducibility and experimentation.

LLM RLF Recipes & Techniques

Recipe 1: Combining Preference Scores with Constraint Penalties

Goal: Fine-tune an LLM using a primary preference-model score while penalizing constraint violations (toxicity, verbosity).

Concept: Use one BaseReward component to extract your external preference score (via InfoValueReward) and add penalty components (e.g., ToxicityPenalty, LengthPenalty). Combine them via RewardComposer.

Example reward_config.yaml:

composer_settings:
  normalize: false

reward_components:
  preference_score:
    class: InfoValueReward
    params:
      info_key: "preference_score"
      default_value: 0.0
  toxicity_penalty:
    class: ToxicityPenalty
    params:
      penalty_value: -2.0
      toxicity_threshold: 0.9
  length_penalty:
    class: LengthPenalty
    params:
      max_length: 256
      penalty_per_token: -0.01

reward_shaping:
  preference_score:
    initial_weight: 1.0
    decay_schedule: none
  toxicity_penalty:
    initial_weight: 1.0
    decay_schedule: none
  length_penalty:
    initial_weight: 0.5
    decay_schedule: linear
    decay_steps: 10000
    min_weight: 0.1

Implementation Notes: Custom components (e.g., ToxicityPenalty, LengthPenalty) need access to generated text and info in their __call__ methods.

Recipe 2: Curriculum Learning for Instruction Complexity

Goal: Train an LLM to follow increasingly complex instructions by adjusting reward weights over time.

Concept: Start with high weight on simple-formatting rewards and low weight on complex reasoning. Gradually shift weights using RewardShaper.

Example reward_config.yaml:

reward_components:
  formatting_check:
    class: FormattingReward
    params:
      reward_value: 1.0
  reasoning_quality:
    class: ReasoningScoreReward
    params:
      info_key: "reasoning_eval_score"
      default_value: 0.0

reward_shaping:
  formatting_check:
    initial_weight: 5.0
    decay_schedule: linear
    decay_steps: 20000
    min_weight: 0.5
  reasoning_quality:
    initial_weight: 1.0
    decay_schedule: linear_increase
    start_step: 5000
    decay_steps: 20000
    max_weight: 5.0

Implementation Notes: You may need custom schedules (e.g., linear_increase) in rllama.rewards.shaping.

Recipe 3: Normalizing Diverse Reward Signals

Goal: Combine signals with different scales without one dominating.

Concept: Specify a normalization_strategy (e.g., z_score, min_max) in composer_settings. Once supported, RewardComposer will normalize raw values before weighting.

Example reward_config.yaml:

composer_settings:
  normalization_strategy: z_score

reward_components:
  # ... preference_score, toxicity_penalty, etc. ...

reward_shaping:
  preference_score:
    initial_weight: 1.0
    decay_schedule: none
  toxicity_penalty:
    initial_weight: 1.5
    decay_schedule: none

Considerations: Normalization introduces non-stationarity. Parameters like norm_window and norm_epsilon will matter when implemented.

Recipe 4: Potential-Based Shaping for Smooth Guidance (Advanced)

Goal: Provide dense guidance via a potential function Φ, preserving the optimal policy.

Concept: Define Φ(state) (e.g., embedding similarity). Compute shaping reward F = γ·Φ(next_state) – Φ(state) and add to the main reward.

Example reward_config.yaml:

reward_components:
  base_reward:
    class: InfoValueReward
    params:
      info_key: "preference_score"
  pbrs_guidance:
    class: LLMPotentialBasedReward
    params:
      potential_function: "embedding_similarity"
      gamma: 0.99
      potential_params:
        target_embedding: [ ... ]

reward_shaping:
  base_reward:
    initial_weight: 1.0
    decay_schedule: none
  pbrs_guidance:
    initial_weight: 0.1
    decay_schedule: none

Implementation Notes: Implement a LLMPotentialBasedReward that calculates Φ from the LLM output.

Recipe 5: Optimizing Reward Hyperparameters with Bayesian Optimization

Goal: Automatically tune initial_weight, decay_rate, normalization_strategy, etc., to maximize alignment metrics.

Concept: Use rllama.rewards.optimization.BayesianRewardOptimizer with Optuna. Define a search space over your YAML config and an objective that runs PPO fine-tuning and returns an evaluation metric.

Example Search Space (Python):

search_space = {
    "composer_settings": {
      "normalization_strategy": {
        "type": "categorical",
        "choices": [None, "z_score", "min_max"]
      }
    },
    "reward_shaping": {
      "preference_score": {
        "initial_weight": { "type": "float", "low": 0.5, "high": 2.0 }
      },
      "toxicity_penalty": {
        "initial_weight": { "type": "float", "low": 0.1, "high": 5.0, "log": True }
      }
    }
}

Example Run Script Snippet:

from rllama.rewards.optimization import BayesianRewardOptimizer

optimizer = BayesianRewardOptimizer(
    base_config=base_config,
    search_space=search_space,
    objective_function=objective,
    objective_kwargs={
      "base_config": base_config,
      "search_space": search_space,
      "component_registry": component_registry
    },
    n_trials=50,
    study_name="llm_reward_tuning",
    storage="sqlite:///llm_tuning.db",
    direction="maximize"
)

best_params, best_value, study = optimizer.optimize()
print(f"Best value: {best_value}")
print(best_params)

After running, analyze results with Optuna visualizations or optuna-dashboard.
