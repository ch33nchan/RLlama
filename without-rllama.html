Was there a collision? Did the agent make progress but use too much energy? Is the time penalty dominating the reward? Without additional logging code, these questions are hard to answer.

  1. Difficult Maintenance and Iteration Adding a new reward component requires:

Modifying the core reward function Carefully balancing it against existing components Potentially retuning all weights Retraining the agent completely 4. Repeated code Inconsistent implementations Time wasted solving the same problems 5. Challenging Debugging When an agent behaves unexpectedly:

The Context Object The context parameter passed to the compute method is a dictionary containing all the information needed to calculate rewards. Common keys include:

Example: Simple Text Length Reward Let’s create a simple reward component that rewards text of a certain length:

Python class TextLengthReward(BaseReward): def init(self, target_length=100, tolerance=20, max_reward=1.0): super().__init__() self.target_length = target_length self.tolerance = tolerance self.max_reward = max_reward

def compute(self, context):
    # Extract the text from the context
    if "response" not in context:
        return 0.0  # No reward if response is missing
    
    text = context["response"]
    length = len(text.split())  # Simple word count
    
    # Calculate how close the length is to the target
    distance = abs(length - self.target_length)
    
    # If within tolerance, give proportional reward
    if distance <= self.tolerance:
        return self.max_reward * (1 - distance / self.tolerance)
    else:
        return 0.0

Component Composition You can create meta-components that combine other components:

Python class CompositeReward(BaseReward): def init(self): super().__init__() self.components = []
